Free PL/SQL Utilities

plstok: A PL/SQL Tokenizer

Plstok is not a separate program -- it is a collection of routines for recognizing lexical tokens within PL/SQL source code. It is the core of the utilities offered at this site. Because it is open software, you can modify or enhance it, or you can incorporate it into your own programs.

Plstok is currently at release level 1.0.0.

Design Features

  1. Each token carries the line number and column number where the corresponding source text occurs.

  2. Plstok can preserve every character of the original source text, including white space and comments. The client code can reconstruct the original source text exactly (in annotated listings, for example).

  3. Optionally, plstok can discard comments and white space so that higher levels of analysis needn't concern themselves with them.

  4. Plstok recognizes tokens of arbitrary length (such as long string literals), subject only to memory constraints.

  5. Through the use of a callback function, the source code may come from a source other than a file (e.g. a database table).

  6. To streamline memory management, plstok caches unused memory blocks for reuse. It releases unneeded memory automatically when memory is exhausted.

  7. Multiple source texts may be parsed concurrently.

  8. Errors are reported in the form of error tokens bearing error messages. The client code chooses how and whether to issue the messages.

  9. Plstok will detect most lexical errors, i.e. malformed tokens, because it can hardly avoid detecting them. An out-of-range numeric literal will escape detection. Syntax errors -- i.e. illegal sequences of legal tokens -- are not its job to detect.

Other Goodies

Included are two layers of software which may be useful in other contexts.

One is a simple memory-management layer which, when compiled in debugging mode, helps to detect memory leaks. It also provides a facility for scavenging memory when resources are low.

The other is a collection of routines for reading generalized source code. It keeps track of line numbers and column numbers, and maintains a small pushback stack for putting characters back into the input stream.

What You Need

  1. util.h: miscellaneous macros and prototypes

  2. sfile.h: header for reading generalized source code

  3. plstok.h: header for plstok routines

  4. plspriv.h: header for internal use within the plstok routines

  5. myassert.c: needed only for compiling in debugging mode

  6. memmgmt.c: routines for memory management

  7. sfile.c: routines for reading generalized source code

  8. plstok00.c: low-level routines internal to plstok

  9. plstok01.c: tokenizing routines

  10. plstok02.c: routines for recognizing reserved words
